## Kritiska buggar

### BUG-001: Infinite spinner om fetchData kastar i invoices/new
**Fil:** `app/dashboard/invoices/new/page.tsx` rad 82–115
**Problem:** `fetchData()` har ingen try/catch/finally. Om något av de tre `Promise.all`-anropen kastar anropas aldrig `setLoading(false)`.
**Symptom:** Oändlig spinner — användaren kan inte skapa faktura.
**Rotorsak:** Asynkron funktion utan felhantering — `loading` fastnar på `true`.
**Fix:**
```typescript
// FEL:
async function fetchData() {
  const [customersApiRes, timeRes, configRes] = await Promise.all([...])
  setLoading(false)
}

// RÄTT:
async function fetchData() {
  try {
    const [customersApiRes, timeRes, configRes] = await Promise.all([...])
  } catch (err) {
    console.error('fetchData error:', err)
  } finally {
    setLoading(false)
  }
}
```

### BUG-003: Child-records raderas utan business_id-filter i DELETE /api/projects — säkerhetsrisk
**Fil:** `app/api/projects/route.ts` rad 468–474
**Problem:** Alla child-deletes saknar `.eq('business_id', business.business_id)`. En autentiserad användare som känner till ett `project_id` tillhörande ett annat företag kan radera dess underdata (milestones, documents, materials m.m.).
**Symptom:** Kritisk dataförlust för drabbat företag.
**Rotorsak:** Säkerhetsfilter saknades vid implementationen.
**Fix:**
```typescript
// FEL:
await supabase.from('project_document').delete().eq('project_id', projectId)
await supabase.from('project_checklist').delete().eq('project_id', projectId)

// RÄTT:
await supabase.from('project_document').delete()
  .eq('project_id', projectId).eq('business_id', business.business_id)
await supabase.from('project_checklist').delete()
  .eq('project_id', projectId).eq('business_id', business.business_id)
// (appliceras på alla child-deletes)
```

### BUG-006: Canvas brush-inställningar uppdateras inte vid verktygs-/färgbyte
**Fil:** `components/project/ProjectCanvas.tsx` rad 127
**Problem:** `useEffect` med dependency `[fabricLoaded]` sätter `freeDrawingBrush.color` och `freeDrawingBrush.width` vid init men uppdateras aldrig när `color` eller `strokeWidth` ändras.
**Symptom:** Penselfärg och tjocklek uppdateras inte när användaren byter verktyg/färg.

```typescript
  canvas.freeDrawingBrush.color = color
  canvas.freeDrawingBrush.width = strokeWidth
}, [color, strokeWidth, tool])
```

### BUG-008: Rate-limit store är in-memory — nollställs vid server-restart
**Fil:** `lib/auth.ts` rad 146
**Problem:** Rate-limiting sker via in-memory `Map`. Vid deploy/restart nollställs räknarna och brute-force-skyddet kringgås.
**Symptom:** Brute-force-skydd fungerar inte i produktion.
**Rotorsak:** Koden är flaggad med kommentar "I produktion bör detta vara Redis" men ej åtgärdad.
**Fix:** Implementera Redis-baserat rate-limit via Upstash eller Vercel KV.
