### BUG-001: saveAsTemplate() visar success-toast vid API-fel
**Fil:** `app/dashboard/quotes/new/page.tsx` rad 1185  
**Problem:** `fetch('/api/quote-templates', {...})` — svarets `res.ok` kontrolleras aldrig. Om API:t returnerar 4xx/5xx visas ändå "Mall sparad!" och modalen stängs.  
**Symptom:** Hantverkaren tror att mallen sparades, men ingenting lagrades i databasen. Stör förtroende för systemet.  
**Rotorsak:** `await fetch(...)` returnerar en `Response` men koden läser varken `.ok` eller `.status`.  
**Fix:**
```typescript
// FEL:
await fetch('/api/quote-templates', { method: 'POST', ... })
toast.success('Mall sparad!')

// RÄTT:
const res = await fetch('/api/quote-templates', { method: 'POST', ... })
const data = await res.json()
if (!res.ok) {
  toast.error(data.error || 'Kunde inte spara mallen')
  return
}
toast.success('Mall sparad!')
```

### BUG-002: Four-eyes-kontroll kringgås när budget_amount skickas i body
**Fil:** `app/api/projects/route.ts` rad 357–399  
**Problem:** `projectValue = body.budget_amount || 0` — om anroparen skickar `budget_amount > 0` i body hoppas hela four-eyes-blocket över (`if (!projectValue)` är false). Projektet kan stängas utan godkännande oavsett värde.  
**Symptom:** Projektstängning på t.ex. 200 000 kr kräver ingen godkännandeprocess om body innehåller `budget_amount`.  
**Rotorsak:** Kontrollen hämtar bara DB-värde som fallback, men jämför inte `body.budget_amount` mot tröskeln.  
**Fix:**
```typescript
// FEL:
const projectValue = body.budget_amount || 0
if (!projectValue) {
  // hämta från DB, kolla tröskel
}

// RÄTT:
let projectValue = body.budget_amount || 0
if (!projectValue) {
  const { data: existingProject } = await supabase
    .from('project')
    .select('budget_amount')
    .eq('project_id', project_id)
    .single()
  projectValue = existingProject?.budget_amount || 0
}
if (
  fourEyesConfig?.four_eyes_enabled &&
  projectValue >= (fourEyesConfig.four_eyes_threshold_sek || 50000)
) {
  // skapa pending_approval och returnera 200 requires_approval
}
```

### BUG-003: api/leads — GET och POST saknar try/catch
**Fil:** `app/api/leads/route.ts` rad 6 (GET) och 144 (POST)  
**Problem:** GET-handlens statistikberäkning (for-loop over `allLeads`) och POST-handlern (`supabase.rpc(...)`) saknar try/catch. Ett oväntat fel kastar ett okantrollerat undantag och returnerar 500 utan felmeddelande.  
**Symptom:** Pipeline-sidan kan krascha tyst med 500 utan loggad error. Svår att felsöka i produktion.  
**Rotorsak:** Rutterna saknar `try { ... } catch (error: any) { return NextResponse.json({ error: error.message }, { status: 500 }) }`.  
**Fix:**
```typescript
// Omslut GET-handlern (rad 6–84) och POST-handlern (rad 144–161) med:
export async function GET(request: NextRequest) {
  try {
    // ... befintlig kod
  } catch (error: any) {
    console.error('Get leads error:', error)
    return NextResponse.json({ error: error.message }, { status: 500 })
  }
}
```
