## Kritiska buggar

### BUG-001: `.single()` utan null-check kraschar payroll-export
**Fil:** `app/api/time-reports/payroll-export/route.ts` rad 38–42
**Problem:** `.single()` på `business_config` utan null-check efter anropet
**Symptom:** API kraschar med "Cannot read property of undefined" om config saknas
**Rotorsak:** `.single()` returnerar `{ data: null, error }` vid ingen träff, men koden antar data alltid finns
**Fix:**
```typescript
// FEL:
const { data: config } = await supabase.from('business_config').select('*').eq('business_id', businessId).single()

// RÄTT:
const { data: config, error: configError } = await supabase.from('business_config').select('*').eq('business_id', businessId).single()
if (configError || !config) {
  return NextResponse.json({ error: 'Business config not found' }, { status: 404 })
}
```

### BUG-002: Saknat business_id-filter på ilike-queries läcker data
**Fil:** `app/api/voice/execute/route.ts` rad 67, 184, 304
**Problem:** `ilike('name', '%...%')` utan föregående `.eq('business_id', businessId)` — returnerar data från ALLA företag
**Symptom:** Sökning kan returnera kunder/projekt från andra företag
**Rotorsak:** business_id-filter saknas på sök-queries
**Fix:**
```typescript
// FEL:
.ilike('name', `%${name}%`)

// RÄTT:
.eq('business_id', businessId)
.ilike('name', `%${name}%`)
```

### BUG-003: `.single()` utan null-check i weekly time-report
**Fil:** `app/api/time-reports/weekly/route.ts` rad ~80–84
**Problem:** Samma mönster som BUG-001 — `business_config` hämtas med `.single()` utan validering
**Symptom:** API returnerar felaktiga default-värden eller kraschar om config saknas
**Rotorsak:** `.single()` + ingen error/null-check
**Fix:** Samma mönster som BUG-001

## Viktiga buggar

### BUG-004: Race-condition i invoices/new/page.tsx useEffect
**Fil:** `app/dashboard/invoices/new/page.tsx` rad 81–95
**Problem:** `fetchData()` i useEffect utan abort/cleanup — om komponenten unmountar mitt i fetch sätts state på unmountad komponent
**Symptom:** React warning + potentiellt felaktiga data om komponenten re-mountas snabbt
**Rotorsak:** Ingen `isMounted`-guard eller AbortController
**Fix:**
```typescript
useEffect(() => {
  let isMounted = true
  const fetchData = async () => {
    const result = await fetch(...)
    if (isMounted) setCustomers(result)
  }
  fetchData()
  return () => { isMounted = false }
}, [business.business_id])
```

### BUG-005: `.single()` på quotes returnerar null utan 404
**Fil:** `app/api/quotes/route.ts` rad 27–32
**Problem:** Om quote inte hittas returneras `data: null` utan att frontend får 404
**Symptom:** Frontend visar tom sida istället för felmeddelande
**Rotorsak:** Felkoll på `error` men inte på `!data`
**Fix:**
```typescript
if (quoteError) throw quoteError
if (!quote) {
  return NextResponse.json({ error: 'Quote not found' }, { status: 404 })
}
```

### BUG-006: Fabric.js-laddningsfel ger ingen UI-feedback
**Fil:** `components/project/ProjectCanvas.tsx` rad 56–66
**Problem:** Dynamic import av fabric.js fångas i `.catch()` som sätter `fabricError = true` men visar ingen synlig felmeddelande för användaren
**Symptom:** Canvas försvinner/spinner försvinner utan förklaring
**Rotorsak:** Error swalloweas tyst
**Fix:** Rendera ett felmeddelande när `fabricError === true` i JSX

### BUG-007: User input direkt i ilike-query utan escaping
**Fil:** `app/api/agent/trigger/tool-router.ts` rad ~873
**Problem:** `params.job_type` används direkt i `.ilike()` utan att escapa `%`-tecken
**Symptom:** Input med `%` kan ge oväntade SQL-resultat
**Rotorsak:** Template-literal utan sanitering
**Fix:**
```typescript
const escapedJobType = (params.job_type as string).replace(/%/g, '\\%')
q = q.ilike('job_type', `%${escapedJobType}%`)
```
